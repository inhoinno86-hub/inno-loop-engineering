내가  프로젝트에 대해서 작업하고자 방향을 보면 아래와 같이 수행하고자 해.



# 프로젝트 생성 및 초안 정의 단계

1. 프로젝트에 대한 전체적인 개요 및 목표를 설정
2. 정의된 개요와 목표를 기반으로, AI에게 내용 정리 및 검토, 제안/의견 수렴 및 결정을 통해 프로젝트의 전체 그림을 재설정
3. 프로젝트에 대한 전체적인 그림이 firm하게 정의되면, 그 내용을 기반으로 프로젝트 개발 로드맵을 생성
4. 생성된 개발 로드맵을 기준으로, 개발 계획을 superpowers의 계획 범주(planning side)의 스킬을 사용하여 계획을 정리



# 프로젝트 개발 진행 단계

1. 프로젝트 생성 및 초안 정의 단계에서 정의한 개발 로드맵 및 개발 계획을 기반으로 세부 개발 계획을 재 정의함
2. 세부 개발 계획은 superpowers의 계획 범주의 스킬을 기준으로 생성
3. 생성된 개발 계획을 근거로, superpowers의 실행 범주(excution side)의 스킬을 사용하여 작업 내용 및 workflow, agent 구성등을 검토하며, make-prompts 스킬을 통해 실제로 작업을 수행할 명령 프롬프트를 단계별로 생성
4. 생성된 단계별 수행 프롬프트를 기반으로, main.md를 기준으로 exec-prompts 스킬을 사용하여 개발 계획에 대한 상세 작업들을 수행함
5. 수행된 작업들에 대해, codex를 통해 결과 피드백을 받고, 그 결과를 기준으로 다시 수정/보완하는 작업 계획을 정의하고, 이에 대해 다시 명령 프롬프트 생성하여, 생성된 명령 프롬프트를 수행하는 loop를 반복함
6. loop는 개발 로드맵 및 프로젝트 초안 정의 단계에서 정의한 개발 계획에 근거하여 goal을 점검하고, 목표가 완수될 때까지 반복 작업을 수행함
7. loop가 반복되는 중간에 필요하다면, 사용자에 명시적 요청에 의해, 중간 점검을 통해 사용자에게 최대한 알기 쉽게 현황 파악 자료를 제공함



상기와 같은 프로젝트 개발 구조를 갖고 프로젝트를 구성하여 진행하고 싶어.

상기 정의에 대해서 좀 더 다듬었으면 좋겠고, 다듬어진 내용을 기반으로, 전역에 agent teams 구성 및 loop 구성을 하고 싶어.
