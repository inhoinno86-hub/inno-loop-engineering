시작 전어느 폴더에서나4-loop 흐름승인과 BLOCKED다른 Codex 세션점검

시작 전

현재 플러그인 소스는 이 저장소의 plugin/inno-loop-engineering에 있습니다. 각 대상 프로젝트에는 상태 파일 .inno-loop/state.json이 생성됩니다. 따라서 여러 프로젝트를 동시에 다뤄도 상태는 섞이지 않습니다.

설치됨: 플러그인은 개인 personal marketplace에 등록되어 있고 Codex에 설치되어 있습니다. 새 Codex 세션에서 $inno-loop-* skill 이름으로 사용할 수 있습니다.

다른 프로젝트·폴더에서 시작하기

플러그인이 설치되기 전에도, 이 저장소에 접근할 수 있다면 아래처럼 CLI를 직접 호출할 수 있습니다. PROJECT_ROOT만 바꾸면 됩니다.

PLUGIN_ROOT=/home/inno/repo/inno-loop-engineering/plugin/inno-loop-engineering
PROJECT_ROOT=/path/to/another-project

# intent.md의 내용은 데이터로 전달합니다.
python3 "$PLUGIN_ROOT/scripts/loopctl.py" \
  --project-root "$PROJECT_ROOT" \
  init --intent "$(cat "$PROJECT_ROOT/intent.md")" --source-ref intent.md

# 현재 상태 확인 (read-only)
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" status

init은 빈 입력과 과도하게 큰 입력을 거부합니다. 다른 프로젝트에서 다시 시작하려면 새 PROJECT_ROOT로 init을 실행하세요.

4-loop 흐름

01project-init
02project-plan
03project-run
04project-review
# init 이후, 근거를 남기며 순서대로 전이합니다.
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" plan --evidence charter-design-roadmap
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" run --evidence plan-v1
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" review --evidence test-report

# 모든 수용 기준이 통과했을 때만 완료합니다.
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" review-complete --evidence review-pass
상태다음 행동남기는 근거
project-init의도·가정·목표 확정 후 plancharter/design/roadmap
project-plan작업·검증·예산 동결 후 runplan hash, validation matrix
project-run변경·테스트 근거 후 reviewrun log, command/test evidence
project-reviewcomplete, replan, blocked, defercriterion 판정과 remediation

승인과 BLOCKED

외부/비가역 작업, 보안·개인정보·비밀정보, 한도 초과, intent 밖 범위·핵심 아키텍처 변경, 반복 평가 실패는 자동 진행하지 않습니다.

# 실제 외부 행동을 하지 않습니다. 승인 요청만 상태에 기록합니다.
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" \
  request-approval --category external_irreversible \
  --action publish --impact public --alternatives "do not publish" --decision approve

# 기록된 사람 승인 근거가 있을 때 재개합니다.
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" \
  resume --evidence approval-ticket-123
무응답은 승인 아님. 불확실한 위험도 BLOCKED로 남겨야 합니다.

다른 Codex 세션에서 skill로 사용하기

플러그인은 개인 personal marketplace에 설치되었습니다. 새 Codex 세션을 열면 skill을 자동 발견할 수 있습니다.

전체 loop: 대상 프로젝트에서 intent.md를 기준으로 inno-loop 수행 부탁해라고 요청하거나 $inno-loop을 사용하세요. intend.md도 별칭으로 허용하지만 둘 다 있으면 안전하게 BLOCKED됩니다.
  • $inno-loop — init, plan, run, review, replan을 COMPLETE 또는 BLOCKED까지 수행
  • $inno-loop-init — 검증된 intent로 상태 생성
  • $inno-loop-plan — init 근거를 plan으로 전이
  • $inno-loop-run — 계획 근거로 실행 단계 시작
  • $inno-loop-review — review 진입, complete 또는 replan
  • $inno-loop-status — 상태를 읽기 전용으로 확인

새 세션에서도 프로젝트별 PROJECT_ROOT만 지정하면 상태는 서로 분리됩니다. 설치 전 또는 shell 자동화가 필요하면 위 CLI 방식을 사용하세요.

점검과 문제 해결

  • status로 현재 loop와 outcome을 먼저 확인합니다.
  • BLOCKED라면 block reason과 evidence를 확인하고, 자동 복구 가능 여부를 판단합니다.
  • 현재 수용 기준 미달은 replan --evidence ...으로만 plan에 되돌립니다.
  • 비차단·저우선 항목만 defer --impact ... --rationale ... --owner ... --revisit-trigger ...로 보류합니다.
# 현재 criterion이 통과하지 않아 계획으로 되돌릴 때
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" \
  replan --evidence criterion-3-failed

# 비차단 backlog를 명시적으로 보류할 때
python3 "$PLUGIN_ROOT/scripts/loopctl.py" --project-root "$PROJECT_ROOT" \
  defer --impact low --rationale "next release" --owner team --revisit-trigger next-release
